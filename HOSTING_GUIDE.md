# دليل استضافة بوت Telegram-Monitor مجاناً على Render

## 1. تجهيز الكود المصدري ورفعه على GitHub

تعتمد منصة Render على سحب الكود مباشرة من مستودعات Git (مثل GitHub). لذا، فإن الخطوة الأولى والأكثر أهمية هي تجهيز الكود المصدري لبوت Telegram-Monitor ورفعه إلى مستودع GitHub خاص بك.

### 1.1. إنشاء مستودع GitHub جديد

1.  توجه إلى [GitHub](https://github.com/) وقم بتسجيل الدخول إلى حسابك.
2.  انقر على زر `+` في الزاوية العلوية اليمنى، ثم اختر `New repository`.
3.  أدخل اسماً للمستودع (مثال: `Telegram-Monitor-Bot`).
4.  اجعل المستودع `Public` أو `Private` حسب تفضيلك (Render يمكنه الوصول لكلاهما، ولكن `Public` أسهل للمبتدئين).
5.  (اختياري) أضف ملف `README.md` و `.gitignore`.
6.  انقر على `Create repository`.

### 1.2. رفع الكود المصدري للبوت إلى GitHub

بعد إنشاء المستودع، ستحتاج إلى رفع ملفات البوت إليه. يمكنك القيام بذلك باستخدام سطر الأوامر (Git Bash أو Terminal):

1.  افتح Terminal أو Git Bash على جهازك.
2.  انتقل إلى مجلد البوت الذي قمت بفك ضغطه سابقاً:
    ```bash
    cd /home/ubuntu/telegram_monitor/Telegram-Monitor-main
    ```
3.  قم بتهيئة المستودع المحلي وربطه بمستودع GitHub الخاص بك:
    ```bash
    git init
    git add .
    git commit -m "Initial commit of Telegram Monitor Bot"
    git branch -M main
    git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPOSITORY_NAME.git
    git push -u origin main
    ```
    **ملاحظة:** استبدل `YOUR_USERNAME` باسم المستخدم الخاص بك على GitHub و `YOUR_REPOSITORY_NAME` باسم المستودع الذي أنشأته.

### 1.3. تعديل ملف `config.py` (هام جداً)

لضمان أن البوت يعمل بشكل صحيح مع متغيرات البيئة في Render، يجب تعديل ملف `src/config.py` ليقرأ متغيرات البيئة بشكل صحيح. حالياً، يستخدم البوت ملف `.env` و `config.json`، ولكن في بيئة Render، سنعتمد بشكل أساسي على متغيرات البيئة التي يتم حقنها مباشرة.

افتح ملف `/home/ubuntu/telegram_monitor/Telegram-Monitor-main/src/config.py` وقم بإجراء التعديلات التالية:

```python
import os
import json
from dotenv import load_dotenv

# Load environment variables from .env file (for local development only)
# load_dotenv()

# Load values from environment variables
API_ID = int(os.getenv('API_ID'))
API_HASH = os.getenv('API_HASH')
BOT_TOKEN = os.getenv('BOT_TOKEN')
CHANNEL_ID = int(os.getenv('CHANNEL_ID')) # تأكد من تحويلها إلى عدد صحيح
SESSION_NAME = os.getenv('SESSION_NAME', 'telegram_monitor_session') # قيمة افتراضية

# Load configuration from config.json (if needed for dynamic settings)
CONFIG_FILE = 'config.json' # تم تعديل المسار ليتناسب مع الجذر الرئيسي للمشروع

def load_json_config():
    if os.path.exists(CONFIG_FILE):
        with open(CONFIG_FILE, 'r') as f:
            return json.load(f)
    return {"TARGET_GROUPS": [], "KEYWORDS": [], "IGNORE_USERS": []} # إرجاع قاموس فارغ إذا لم يكن الملف موجوداً

def update_json_config(config):
    with open(CONFIG_FILE, 'w') as f:
        json.dump(config, f, indent=4)

# Load dynamic config from JSON
json_config = load_json_config()

TARGET_GROUPS = json_config.get('TARGET_GROUPS', [])
KEYWORDS = json_config.get('KEYWORDS', [])
IGNORE_USERS = json_config.get('IGNORE_USERS', [])
```

**الشروحات الهامة للتعديلات:**
- تم التعليق على `load_dotenv()` لأن Render سيوفر المتغيرات مباشرة.
- تم تغيير `CHANNEL_ID` ليتم تحويله إلى `int` مباشرة من متغير البيئة.
- تم تعديل `SESSION_NAME` ليكون له قيمة افتراضية.
- تم تعديل مسار `CONFIG_FILE` إلى `config.json` مباشرة، واضيفت معالجة لحالة عدم وجود الملف.

بعد إجراء هذه التعديلات، قم برفع التغييرات إلى مستودع GitHub الخاص بك:

```bash
git add src/config.py
git commit -m "Update config.py for Render environment variables"
git push origin main
```

الآن، أصبح الكود الخاص بك جاهزاً للاستضافة على Render.

## 2. إعداد دليل الاستضافة على Render خطوة بخطوة

Render هي منصة سحابية توفر استضافة مجانية لمشاريع الويب والخدمات الخلفية، وهي خيار ممتاز لاستضافة بوتات تيليجرام. اتبع الخطوات التالية لنشر البوت الخاص بك:

### 2.1. إنشاء حساب على Render وتسجيل الدخول

1.  توجه إلى موقع [Render](https://render.com/) وقم بإنشاء حساب جديد أو تسجيل الدخول باستخدام حساب GitHub الخاص بك.
2.  بعد تسجيل الدخول، ستنتقل إلى لوحة التحكم الخاصة بك.

### 2.2. إنشاء خدمة جديدة (New Web Service)

1.  في لوحة تحكم Render، انقر على `New` ثم اختر `Web Service`.
2.  ستُطلب منك ربط حساب GitHub الخاص بك بـ Render إذا لم تكن قد فعلت ذلك بالفعل. اتبع التعليمات لمنح Render الأذونات اللازمة للوصول إلى مستودعاتك.
3.  اختر المستودع الذي يحتوي على كود بوت Telegram-Monitor (الذي قمت برفعه في الخطوة السابقة).

### 2.3. إعدادات الخدمة (Service Settings)

بعد اختيار المستودع، ستحتاج إلى تكوين إعدادات الخدمة:

-   **Name:** أدخل اسماً لخدمتك (مثال: `telegram-monitor-bot`).
-   **Region:** اختر أقرب منطقة جغرافية إليك أو إلى جمهور البوت الخاص بك.
-   **Branch:** اتركها `main` (أو الفرع الذي يحتوي على الكود الخاص بك).
-   **Root Directory:** اتركها فارغة إذا كان الكود في الجذر الرئيسي للمستودع، أو حدد `Telegram-Monitor-main` إذا كان الكود داخل هذا المجلد الفرعي (كما هو الحال في مشروعنا).
-   **Runtime:** اختر `Python 3`.
-   **Build Command:** أدخل الأمر التالي لتثبيت الاعتمادات:
    ```bash
    pip install -r requirements.txt
    ```
-   **Start Command:** أدخل الأمر التالي لتشغيل البوت:
    ```bash
    python3 src/main.py
    ```
    **ملاحظة:** تأكد من أن المسار `src/main.py` صحيح بالنسبة لموقع ملف `main.py` داخل مشروعك.

### 2.4. اختيار الخطة المجانية (Free Plan)

-   في قسم `Instance Type`، اختر `Free`.
-   انقر على `Create Web Service`.

سيبدأ Render الآن في بناء ونشر البوت الخاص بك. يمكنك متابعة سجلات النشر في لوحة التحكم. قد يستغرق هذا بضع دقائق. إذا كانت هناك أي أخطاء في البناء أو التشغيل، ستظهر في السجلات.

**ملاحظة هامة:** في هذه المرحلة، قد يفشل البوت في التشغيل بسبب عدم وجود متغيرات البيئة المطلوبة (API_ID, API_HASH, BOT_TOKEN, CHANNEL_ID). هذا أمر طبيعي وسنقوم بضبطها في الخطوة التالية.

### 3.3. إعادة نشر الخدمة (Redeploy Service)

بعد إضافة جميع متغيرات البيئة، ستحتاج إلى إعادة نشر الخدمة لتطبيق التغييرات:

1.  في صفحة الخدمة الخاصة بك في Render، انقر على زر `Manual Deploy` أو `Deploy latest commit`.
2.  سيقوم Render بإعادة بناء وتشغيل البوت الخاص بك مع متغيرات البيئة الجديدة.

راقب سجلات النشر (Logs) للتأكد من أن البوت بدأ العمل بنجاح. إذا واجهت أي مشاكل، تحقق من قيم متغيرات البيئة وتأكد من صحتها.

## 4. نصائح للحفاظ على استمرارية تشغيل البوت 24/7 مجاناً

منصة Render في خطتها المجانية قد تقوم بإيقاف الخدمة مؤقتاً إذا لم تتلقَ أي طلبات HTTP لفترة زمنية معينة (Sleep Mode). للحفاظ على تشغيل البوت بشكل مستمر، يمكنك اتباع النصائح التالية:

### 4.1. استخدام خدمة مراقبة خارجية (Uptime Monitor)

يمكنك استخدام خدمات مثل [UptimeRobot](https://uptimerobot.com/) أو [Cron-job.org](https://cron-job.org/) لإرسال طلب HTTP إلى رابط الخدمة الخاص بك في Render كل 5 أو 10 دقائق. هذا سيمنع الخدمة من الدخول في وضع السكون.

### 4.2. إضافة خادم ويب بسيط داخل البوت

لجعل البوت يستجيب لطلبات HTTP، يمكنك إضافة خادم ويب بسيط باستخدام مكتبة `Flask` أو `FastAPI` داخل كود البوت.

1.  أضف `flask` إلى ملف `requirements.txt`.
2.  أضف الكود التالي في ملف `src/main.py`:

```python
from flask import Flask
from threading import Thread

app = Flask('')

@app.route('/')
def home():
    return "I'm alive!"

def run():
  app.run(host='0.0.0.0',port=8080)

def keep_alive():
    t = Thread(target=run)
    t.start()
```
